# UI95 Editor Audit (2026-05-02)

## Что уже хорошо
- Выбран **relative layout** (`nx/ny/nw/nh`) — это правильная база для резинового UI.
- Есть `pivot` + `anchor` — это уже задел под полноценный layout-engine.
- Структура редактора понятная: hierarchy / viewport / inspector / export.

## Главные проблемы
1. **Смешение относительных и абсолютных слоёв логики**
   - В одном месте система «про относительность», в другом — комментарии и мета в пикселях.
   - Из-за этого экспорт был шумным и противоречивым.

2. **Экспортный DSL пока слабоспецифицирован**
   - Нет версионирования формата (`@format_version`).
   - Нет явного разделения обязательных и опциональных полей.
   - Нет нормальной валидации диапазонов (`0..1`) при импорте.

3. **Операции редактирования без транзакций**
   - Undo есть, но лучше фиксировать изменения как атомарные операции (begin/end op).

4. **Слишком монолитный `base.html`**
   - UI, рендер, парсер, сериализация, события в одном файле.

## Что изменено сейчас
- Из export удалён вывод размера виртуального canvas (`@virtual_size = W x H`).
- Из export удалён служебный комментарий с px (`; @WxH px=... sz=...`).
- Обновлён комментарий заголовка: теперь явно сказано, что координаты относительные и размер canvas намеренно не экспортируется.

## Рекомендованная эволюция синтаксиса (DSL)

```ini
@format_version = 1
@ui_file        = main_menu
@namespace      = game.ui
@theme          = win98_classic

window_main:
    class        = WINDOW
    caption      = Main Menu
    pos          = 10%, 8%
    size         = 80%, 84%
    pivot        = 0.5, 0.5
    anchor_tl    = 0, 0
    anchor_br    = 1, 1
```

### Обязательные поля
- `class`, `pos`, `size`

### Опциональные
- `caption`, `pivot`, `anchor_*`, `style_class`, `events`

## Вопросы к тебе (важно для следующего шага)
1. Нужна ли **обратимая совместимость** со старыми файлами, где есть `@virtual_size`?
2. Нужен ли экспорт в **двух режимах**:
   - `clean` (только relative),
   - `debug` (с px-комментариями)?
3. Оставляем проценты (`10%`) как основной стиль, или пишем голые `0.1`?
4. Нужен ли в DSL блок `constraints` (min/max size), как в layout-системах?
